初探 Vapor - 安裝

2018-11-11 Kommentare

偷懶許久，默默回首發現上一篇已是 2018 年初，呵呵

今天就來談談 Server-side-Swift 相關的入門知識

iOS 開發者在開發 APP 時利用後端大大們開好的 API 獲取需要的資訊想必是相當常見

身為一名使用 swift 進行開發的菜雞

做什麼都先想看看能不能用 swift 解決我想是很正常的

為此， Vapor 就是個不錯的選擇

官網提供不錯的文檔，網路上資源也相當豐富

目前還有免費的 cloud 平台讓我們能輕鬆的架設服務

首先，我們先來到 Vapor 的官網看看

Vapor

安裝相當的簡單

首先需要安裝 9.3 版以上的 Xcode (會想用 vapor 但不裝 Xcode 的人應該不多吧ＸＤ)

以及 4.1 版以上的 swift

swift 版本可以打開 termaial 輸入

進行確認

接下來就是安裝 Vapor 拉(對，又是 homebrew )

安裝完成輸入

看到幫助訊息就代表安裝成功拉

接下來就讓我們簡單的創建一個 Vapor 應用並部署他吧

我們就來創建一個命名為 vapor-cloud-drw 的專案

首先在終端創建一個 vapor-project 的文件夾並進入該文件夾

接下來創建名叫 vapor-cloud-drw 的 vapor 專案

完成後進入專案資料夾

並建置一下專案

第一次建置系統會提示你沒有 .build 的文件夾，並開始下載需要的文件

首次建置完成之後，為了讓我們能順利的開發我們的 vapor 專案

使用 Xcode 當然是個不錯的選擇

為了讓 Vapor 產生 Xcode 的專案

vapor-cloud-drw 目錄下輸入

系統就會幫你生成 Xcode 專案

加上 -y 則是告訴系統在建置完之後直接幫我們用 Xcode 打開

在 Xcode 打開之後系統會自動開始幫忙索引文件內容，等吧(好想換更新的 mac 啊嗚嗚嗚嗚)

索引完成之後在我們興奮地按下 command + R 試跑 vapor 服務之前

先把左上的建置設定修改一下

OK GO~

執行起來等一下就會看到在 console 系統有提示我們

打開網頁訪問一下這個位置就會發現網頁網頁顯示

告訴你我們可以正式開始探索 vapor 的功能拉～～～
